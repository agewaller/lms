# 08. テスト計画

> **LIVE 環境で発注テストを絶対にしない**。
> すべては `cfgTradingMode="PAPER"` モードで、発注 API をモックしたうえで検証する。

---

## 1. テストモードの実装

`modOrder.SendOrder` の先頭で `cfgTradingMode` を見て、`PAPER` 時は
**発注関数を呼ばずに擬似応答を返す**。

```vb
Public Function SendOrder(ByRef o As clsOrder) As Boolean
    If modConfig.TradingMode() = "PAPER" Then
        SendOrder = SimulateOrder(o)   ' 擬似応答
        Exit Function
    End If
    ' LIVE 経路
    SendOrder = SendOrderLive(o)
End Function
```

擬似応答は：
- `NEW → SENT` 即座
- 一定確率で `FILLED`
- 10% で `REJECT`（エラーコード `SIM_ERR`）
- 10% で `PART`（半数量）

## 2. テスト一覧（単体 → 結合 → 停止条件）

### T01. シート入力テスト（単体）

| 項目 | 内容 |
|---|---|
| 目的 | `設定`, `銘柄一覧` の読み込みが正しいこと |
| 事前条件 | 名前付きセル定義済、`銘柄一覧` に5銘柄 |
| 操作 | `modConfig.Refresh` → `modSignal.ScanAll` |
| 期待 | 5 銘柄すべて走査、不正銘柄は `銘柄一覧.売買禁止フラグ=TRUE` |
| NG 例 | 銘柄名が `#N/A` のまま、走査件数 0 |

### T02. シグナル判定テスト（単体）

| 項目 | 内容 |
|---|---|
| 目的 | `clsStrategyBreakout` が期待通りに HIT するか |
| 事前条件 | テスト用の疑似 `RSS_raw` にブレイク条件を満たす値 |
| 操作 | `modSignal.EvaluateSymbol("7203")` |
| 期待 | `シグナル` に 1 行追加、`状態=WAIT` |
| NG 例 | シグナルが出ない、または重複して複数出る |

### T03. 発注前チェックテスト（単体）

| 項目 | 内容 |
|---|---|
| 目的 | `modRisk.PreCheckOrder` が全条件を検証するか |
| 事前条件 | 各リスク条件を個別に違反させたテストケース 15 個 |
| 操作 | 各ケースで `PreCheckOrder(order)` 呼出 |
| 期待 | 条件ごとに文字列で `NG:reason` が返る |
| NG 例 | `OK` を返してしまう、または関係ない理由で `NG` |

### T04. 疑似発注テスト（結合）

| 項目 | 内容 |
|---|---|
| 目的 | PAPER モードで発注シーケンスが完走 |
| 事前条件 | `cfgTradingMode=PAPER` |
| 操作 | シグナル生成 → 承認ボタン押下 |
| 期待 | `注文管理` に `SENT` 追加、`シグナル.状態=SENT` |
| NG 例 | エラーでブック固まる、状態が中途半端 |

### T05. 一部約定想定テスト

| 項目 | 内容 |
|---|---|
| 目的 | `SENT → PART → FILLED` の遷移 |
| 事前条件 | PAPER モード、シミュレータが半分約定 |
| 操作 | T04 を複数回 |
| 期待 | `注文管理.状態=PART` を経て `FILLED` |
| NG 例 | `PART` のまま停滞、建玉未更新 |

### T06. 取消テスト

| 項目 | 内容 |
|---|---|
| 目的 | `CancelOrder` が正しく動く |
| 事前条件 | T04 で `SENT` の注文が存在 |
| 操作 | 取消ボタン押下 |
| 期待 | `注文管理.状態=CANCEL`, `シグナル.状態=CANCEL` |
| NG 例 | 取消後も約定してしまう、状態が変わらない |

### T07. 緊急停止テスト

| 項目 | 内容 |
|---|---|
| 目的 | KillSwitch 押下で全停止 |

### T08. 再起動復旧テスト

## 3. 回帰テストの自動化

- 戦略の Evaluate は純関数なので **VBA 単体テスト** を書く。
  - `tests/TestStrategyBreakout.bas` に `Public Sub Test_Breakout_1()` 形式で作成
  - 起動時に `cfgRunSelfTests=TRUE` ならすべて実行
- リスクチェックも純関数なので同様。
- シート更新系は `PAPER` モードでの結合テストで確認。
