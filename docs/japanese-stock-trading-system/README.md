# 日本株 自動売買支援システム 設計書

楽天証券「マーケットスピード II RSS（以下 MS2 RSS）」+ Excel/VBA を中核とした、
**日本の上場株式（現物）向け** 半自動売買支援システムの設計ドキュメント一式です。

> 本ドキュメントは「資産の項目（=ポートフォリオ・建玉・余力・損益）」を起点に、
> シグナル発生から人による承認、発注、約定確認、ログ、通知、リスク停止までを
> Excel/VBA で実装可能な粒度で記述します。

---

## 0. 厳守事項（必ず最初に読む）

- 楽天証券の **国内株** は一般的な REST API による自由な発注ができません。
  本設計は **MS2 RSS + Excel/VBA** を前提とし、REST/WebSocket 前提の設計は含みません。
- 本書で「**要確認**」と書いた箇所は、楽天証券の RSS 関数仕様書 / MS2 II の最新マニュアル
  を一次ソースで確認してから実装してください（推測で埋めない）。
- 自動売買には **法令・約款上の制約** があります。
  本書は「ツールでできる/できない」の **技術設計** であり、利用可否の判断は利用者責任です。
- 誤発注防止と損失制御を最優先します。最初から完全自動売買は **目指しません**。

## 2. 前提確認（できること / できないこと / 要確認）

### 2-1. MS2 RSS + Excel/VBA で **できる** こと（一般に知られている範囲）

- **マーケットデータ取得**（RSS 関数経由でセルに取得、一定周期で自動更新）
  - 現在値・前日比・始値/高値/安値・出来高・VWAP
  - 板情報（売気配/買気配 数本〜10本程度、要バージョン確認）
  - 歩み値・四本値（一定範囲）
  - 信用残・出来高加重平均など補助指標
- **口座・建玉・余力情報の取得**
  - 預り金・買付余力・信用新規建可能額（**項目名は要確認**）
  - 現物保有銘柄一覧・数量・平均取得単価・評価損益
  - 信用建玉一覧（拡張時）
- **発注関連**
  - 現物 買い/売り 注文の発行（成行/指値/逆指値、有効期限指定）
  - 注文取消・注文訂正
  - 注文一覧・約定一覧の取得
- **VBAからの操作**
  - RSS関数で取得したセル値を VBA から読む
  - VBAから発注用シートにパラメータを書き込み、RSS関数 or 専用関数経由で発注
  - 約定通知用のセル更新を `Worksheet_Change` でフックしてログに残す

### 2-2. **できない / 苦手** な領域

- **REST APIライクな自由な双方向通信** … 不可。あくまで Excel セル経由
- **マルチプロセス並列発注** … 単一 Excel/VBA インスタンス前提
- **確実なミリ秒レイテンシ** … RSS の更新周期・Excel 計算サイクルに律速
- **マーケットスピード II 自体が落ちた場合の自動リカバリ**
  - MS2 II 本体・RSS の生死確認は **死活監視を自前で実装する必要あり**
- **完全な板再構築（フル板）** … 提供気配段数の範囲のみ
- **過去長期分の高頻度ティックデータ取得** … 別途データ供給源が必要
- **ザラ場以外の発注** … 取引所/楽天証券の受付時間に従う
- **OS再起動・PCスリープからの自動復帰** … 運用回避が必須

### 2-3. **要確認**（実装前に必ず一次ソースで確認すべき項目）

| # | 項目 | 確認理由 |
|---|---|---|
| R1 | RSS 関数の **正確な関数名・引数仕様**（最新版） | バージョンで変わる |
| R2 | **発注用関数 / 発注専用シート** の使い方 | RSS 単体 or 別ツール経由か |
| R3 | 1リクエストあたりの **発注スロットリング上限** | 連打規制・不正注文判定 |
| R4 | **板情報の段数** と更新頻度 | シグナル設計に直結 |
| R5 | 余力・建玉の **項目名（買付余力 / 受渡前余力 等）** の正確な定義 | リスク計算ミス防止 |
| R6 | **指値/成行/逆指値/IOC/不成 等の対応執行条件** | 注文戦略の選択肢 |
| R7 | **注文 ID の払い出し方式**（VBA 側で受け取れるか） | 二重発注防止に必須 |
| R8 | **エラーコード一覧** と再送可否 | エラーフロー設計に必須 |
| R9 | RSS 更新が **停止/凍結したかを検知する手段**（タイムスタンプ/Heartbeat） | 停止条件に必須 |
| R10 | **PTS 取引** の取り扱い（板/発注） | 対象外で良いか確認 |
| R11 | 信用取引の **制度信用 / 一般信用** の発注区分の指定方法 | 将来拡張時 |
| R12 | **約定通知** がプッシュされるか、ポーリングが必要か | ログ精度に直結 |
| R13 | RSS から取得した **保有銘柄の "預り区分"**（NISA/特定/一般）の扱い | 発注時の口座区分指定 |
| R14 | **同時起動 Excel** の制限（複数ブックから RSS を叩けるか） | アーキ設計 |
| R15 | 楽天証券 **約款上の自動売買の取り扱い** | 利用前に必読 |

> 上記 R1〜R15 は **本書では推測で埋めません**。実装着手時に必ず確認し、結果を `docs/japanese-stock-trading-system/RSS_VERIFIED.md` に記録してください。
