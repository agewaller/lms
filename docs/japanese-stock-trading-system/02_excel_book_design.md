# 02. Excel ブック設計

ブック名：`trading.xlsm`
保存先：ローカル固定パス（例：`C:\Trading\trading.xlsm`）。自動バックアップは引け後に Python で別フォルダへコピー。

> 本書のすべての「取得元＝RSS」は **"RSS関数で取得する"** の意味です。
> 実際の関数名・引数は `RSS_VERIFIED.md`（実装時作成）を参照。

---

## 1. シート一覧

| シート名 | 目的 | 種別 |
|---|---|---|
| `設定` | 名前付きセル・定数・閾値 | 設定 |
| `銘柄一覧` | 監視対象銘柄マスター | マスター |
| `シグナル` | 売買候補（承認待ち含む） | 業務 |
| `注文管理` | 発注中/発注済/取消の注文台帳 | 業務 |
| `建玉管理` | 現物保有（将来：信用建玉） | 業務 |
| `余力管理` | 口座余力・NAV・評価損益 | 業務 |
| `約定履歴` | すべての約定（Fill） | 履歴 |
| `売買ログ` | シグナル/発注/取消/理由ログ | 履歴 |
| `エラーログ` | 例外/通信断/値異常 | 履歴 |
| `監視ダッシュボード` | 稼働状態・損益・停止フラグ | UI |
| `RSS_raw`（隠し） | RSS 関数の生取得エリア | データ |
| `_meta`（隠し） | バージョン・起動時刻・ビルド情報 | 管理 |

> 「業務」シートは **行追加/更新が頻繁**、「履歴」シートは **追記専用（INSERT ONLY）**、
> 「データ」シートは **人が触らない** 原則を守る。

## 3. 名前付きセル一覧（VBAから参照）

> 名前付きセルは `設定` シート上の特定セルに対して `名前の定義` で付与する。
> VBA からは `ThisWorkbook.Names("cfgXxx").RefersToRange.Value` で参照する。

| 名前 | 型 | 既定値 | 意味 |
|---|---|---|---|
| `cfgTradingEnabled` | Bool | FALSE | 取引ON/OFF（OFF時は発注を一切行わない） |
| `cfgKillSwitch` | Bool | FALSE | 緊急停止フラグ（TRUEで即全停止） |
| `cfgApprovalRequired` | Bool | TRUE | 承認必須フラグ（Phase3で一部 FALSE 可） |
| `cfgTradingMode` | String | "PAPER" | `PAPER`/`LIVE`/`OBSERVE` |
| `cfgMaxLossPerTrade` | Long | 5000 | 1回最大損失（円） |
| `cfgMaxLossPerDay` | Long | 20000 | 1日最大損失（円） |
| `cfgMaxPositionPerSymbol` | Long | 300000 | 1銘柄あたり最大投資額（円） |
| `cfgMaxConcurrentPositions` | Long | 3 | 同時保有銘柄数上限 |
| `cfgMaxOrdersPerDay` | Long | 10 | 1日最大発注回数 |
| `cfgMinLiquidityVolume` | Long | 100000 | 最小必要出来高 |
| `cfgMaxSpreadBps` | Long | 50 | 最大許容スプレッド(bps) |
| `cfgMinTickInterval` | Long | 500 | 発注間隔の最小ミリ秒（二重クリック防止） |
| `cfgNotifyEnabled` | Bool | TRUE | 通知ON/OFF |
| `cfgNotifySlackWebhook` | String | "" | Slack Webhook URL（本番時のみ） |
| `cfgRssHeartbeatCell` | String | "RSS_raw!A1" | RSS 死活監視セル |
| `cfgRssHeartbeatMaxAgeSec` | Long | 10 | RSS 更新停止と判定する秒数 |
| `cfgLoopIntervalMs` | Long | 3000 | 監視ループの周期(ms) |
| `cfgMarketOpenTime` | Time | 09:00:00 | ザラ場開始 |
| `cfgMarketCloseTime` | Time | 15:00:00 | ザラ場終了 |
| `cfgLunchStartTime` | Time | 11:30:00 | 前場終了 |
| `cfgLunchEndTime` | Time | 12:30:00 | 後場開始 |
| `cfgAllowedOrderType` | String | "LIMIT" | `LIMIT`/`MARKET`/`ANY`（成行禁止時は `LIMIT` 固定）|
| `cfgResumeRequiresManual` | Bool | TRUE | 停止後の再開は手動必須 |
| `stateTodayRealized` | Long | 0 | 本日の実現損益（計算結果） |
| `stateTodayOrders` | Long | 0 | 本日の発注回数 |
| `stateLastOrderTick` | Date | 0 | 最後の発注 Tick |
| `stateHaltReason` | String | "" | 停止中の理由 |

> 名前の接頭辞は `cfg`（設定）/ `state`（状態）/ `btn`（ボタン位置）で統一すること。

## 4. データ保護とシート保護方針

| シート | ユーザ編集 | VBA 編集 | 行追加方針 | 保護 |
|---|---|---|---|---|
| `設定` | 可（特定セルのみ） | 不可 | - | シート保護 + 名前付きセル以外ロック |
| `銘柄一覧` | 可 | 可 | 追加/削除 | 列ロック必須（PK列を誤消ししない） |
| `シグナル` | 承認列のみ | 可 | 追記 | 行削除禁止（アーカイブは引け後） |
| `注文管理` | 不可 | 可 | 追記/更新 | シート保護 |
| `建玉管理` | 不可 | 可 | 再生成 | シート保護 |
| `余力管理` | 不可 | 可 | 追記 | シート保護 |
| `約定履歴` | 不可 | 可 | INSERT ONLY | シート保護 |
| `売買ログ` | 不可 | 可 | INSERT ONLY | シート保護 |
| `エラーログ` | 不可 | 可 | INSERT ONLY | シート保護 |
| `監視ダッシュ` | ボタン押下のみ | 可 | - | シート保護 + ボタン除外 |
| `RSS_raw` | 不可 | 読込のみ | - | 隠し + シート保護 |
| `_meta` | 不可 | 可 | - | 隠し + パスワード付保護 |

## 5. ID 採番ルール

| ID | 形式 | 例 |
|---|---|---|
| シグナルID | `SIG-<YYYYMMDD>-<連番4桁>` | `SIG-20260409-0042` |
| 注文ID（内部） | `ORD-<YYYYMMDD>-<連番4桁>` | `ORD-20260409-0021` |
| 約定ID | `FIL-<注文ID>-<枝番2桁>` | `FIL-ORD-20260409-0021-01` |
| 保有ID | `POS-<コード>-<連番3桁>` | `POS-7203-001` |
| ログID | 連番 Long | 12345 |
| エラーID | 連番 Long | 1 |

- 日付ロールオーバー時に連番リセットを **モーニング初期化で実行**。
- 連番発行は `_meta` シートのカウンタセルを **排他インクリメント**（モジュール: `modUtils.NewId`）。
